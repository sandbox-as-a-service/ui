## Where things live

- **Stories**: colocated with components as `*.stories.tsx` (CSF) + optional `*.mdx` for narratives.

## Authoring stories (CSF)

```tsx
// Button.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import { Button } from "@/components/button";

const meta: Meta<typeof Button> = {
  title: "Components/Button",
  component: Button,
  args: { label: "Continue" },
  parameters: {
    a11y: { disable: false },
    controls: { expanded: true },
  },
};
export default meta;

export type Story = StoryObj<typeof Button>;

export const Primary: Story = {
  args: { variant: "primary" },
};

export const Destructive: Story = {
  args: { variant: "destructive" },
};
```

**Conventions**: Prefer CSF (stories as data), descriptive story names, realistic defaults via `args`. Avoid deep imports

- **Chromatic** (if enabled) for visual regression

---

## Quality gates

- **Accessibility**: stories must pass a11y checks.
- **Tests**: add at least one interaction test for complex states.
- **Docs**: include usage notes and a props table (auto-generated) where helpful.
- **Visuals**: significant UI changes require a reviewed visual diff (Chromatic or equivalent).
